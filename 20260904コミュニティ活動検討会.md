## アジェンダ

### 現状報告
* QGIS4.2 がこの秋にLTRになる予定
* 現状でOSGeo.JPがQGISの日本ローカルコミュニティ。それの連絡先（投票権あり）のメンバーを多分変更する必要がある
  [ローカルユーザーグループ](https://www.qgis.org/ja/community/groups/)
* 2年前にQGIS.org のWEBページの管理方法が変わった（Sphinx →Hugo）この時最初は国際化の実装をしなかったので従来のTransifexを利用した翻訳体制がいったん利用できなくなった（Webサイトについて）しかし、最近Transifexを利用した翻訳リソースを使って国際化された。翻訳率が35%を超える言語はWEBサイトで選択してその言語で表示できるようになった。
  [妥協の無い地理空間情報](https://www.qgis.org/ja/)

* QGIS.jp というドメイン名をMIERUNEさんがおさえているが、どうするか？　他のローカルコミュニティではローカル独自のQGISサイトを作成しているところもある。QGISの本家サイトのHugoのテーマを使って同じような見てくれのローカルコミュニティのサイトをつくっているところもある。


### 翻訳関連報告と課題
* QGIS Desktop
翻訳率100%
ただし時々翻訳の間違いがあるのでレビューが必要

* QGIS Documentation
ビルドをかけるとエラーが表示されるので、時々ビルドしてエラーメッセージを取得、対応するといいかも

* QGIS Website
翻訳率35%を超えないと本家WEBページでその言語が選択に出てこない。
毎日日本時間の午後2時ぐらいに翻訳率チェックのスクリプトがGitHub Actionで動いている・
日本語は最近（6月3日）選択肢にでるようになった。
2年前にHugoというフレームワークにWebsiteが変わった。そのため翻訳用テキスト内にHugoの制御用テキストが混入しているので、それの扱いを間違えるとえHugoのコンパイルが失敗する。（テキストはMarkdownで記述されているが）。以前のサイトはSphinxで管理していたのでreStructured Textで記述されていたため、制御系の文字列がDocumentと同じだったので翻訳者もそれを理解していればよかった。Websiteの翻訳はHugoの制御テキストを理解する必要があるので、それのガイドラインのようなものが必要。
日本語選択状態だと検索ができない。　[Blog post](https://makewithhugo.com/add-search-to-a-hugo-site/) [Gist](https://gist.github.com/eddiewebb/735feb48f50f0ddd65ae5606a1cb41ae) この方法で検索が実装されているらしいがQGISサイトのどこに記述があるのか調査中

* レビューと翻訳内容の論議
Transifexにアカウントを作成している翻訳者は多いが翻訳内容のレビューがあまりなされていない。DeskTopの翻訳で間違いがいくつかあったが、それをチェックしたり論議したりする体制が無い。レビューをする体制と翻訳内容について検討、論議できるようにする必要がある。

* ガイドラインの作成
翻訳ガイドラインが必要　　日本語ガイドライン　reStructured Text ガイドライン　Hugo ガイドライン。　ガイドラインをGithubのWikiかMarkdownのファイルにして共用しておいてTransifexへの参加者が参照したりガイドラインの修正意見をだせたりするようにしておくといいのでは。（以前国内で作成したQGIS Hackfestで作成したことがある [2015 QGIS Hackfest 記録](https://github.com/OSGeo-jp/QGIS/wiki/2015%E5%B9%B4%E3%80%80QGIS-Hackfest%E3%80%80Tokyo%E3%80%80%E3%80%80%E7%BF%BB%E8%A8%B3%E9%96%A2%E4%BF%82%E4%BD%9C%E6%A5%AD%E8%A8%98%E9%8C%B2) ）　[ガイドラインリポジトリ候補](https://github.com/QGIS-jp/translation_guidelines) をつくってみたがここにガイドラインを作成してTransifex参加者に参照してもらうようにしてはどうか
